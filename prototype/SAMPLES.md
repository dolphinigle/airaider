# SAMPLES — morning handoff

A navigation guide to every committed transcript in `prototype/fixtures/`.
Read this first to browse the prototype's actual output without re-running
LLM calls.

All `*-real.json` transcripts were generated by `gpt-4.1-nano` against the
strict JSON-schema scenario contract. All `*-mock.json` transcripts come
from the deterministic MockScenarioLLM and are pinned by snapshot tests.

## Where to look first

If you only read three files, read these:

1. `fixtures/captive-01.captive-real.json` — 5 captive actions (ransom /
   sell / display / recruit / execute) against the same prisoner. Tonally
   distinct outputs per action; demonstrates the captive cycle works.
2. `fixtures/campaign-day-03.real.json` — last day of the 3-day campaign.
   Mercs are visibly fatigued; nano references it in the narration.
3. `fixtures/raid-05-tavern.transcript-real.json` — closes on "the weight
   of unspoken stories thick in the air"; shows backstory shaping tone.

## Campaign (persistent roster) — read these in order

Run as: `npm run day -- fixtures/day-NN.json --real --roster=fixtures/campaign-roster.json`.

| Day | Fixture | Transcript | Final fatigue snapshot (cumulative) |
|---|---|---|---|
| 1 | `day-01.json` | `campaign-day-01.real.json` | Marek 3, others 0–2 |
| 2 | `day-02.json` | `campaign-day-02.real.json` | Marek 4, Veska 3, Dren 2 |
| 3 | `day-03.json` | `campaign-day-03.real.json` | Marek 5, Veska 5, Imogen 3, Dren 3, Roselle 2 |

Final roster state lives at `fixtures/campaign-roster.json` (schemaVersion: 1,
dayCount: 3). Day-3 narrations reference the cumulative wear without being
prompted to — e.g. "cautious approach reveals nothing".

## Generator

`npm run recruit -- --count 3 --seed bog-tavern` produces 3 candidates from
the 22-tag vocabulary at `data/tags.json`. No fixture committed because the
generator is RNG-deterministic on seed; reproduce locally to inspect.

## What is NOT yet shown

- Wounds / permadeath math (HP exists in roster schema but no scenario yet
  inflicts damage).
- Reputation tracking beyond what captive actions write.
- Multi-day quest arcs (each scenario is still atomic).
- Legendary tag `pers:touched-by-the-mire` only exists in the vocabulary;
  no merc carries it yet.
- GUI of any kind. Prototype is console-only by design.

## How to regenerate (cheap & safe)

```sh
# Mocks only — instant, no LLM cost
npm test
npm run scenario -- fixtures/raid-01.json --out fixtures/raid-01.transcript-mock.json
npm run day -- fixtures/day-01.json --out fixtures/day-01.day-mock.json

# Real nano (costs cents; never commits the API key)
npm run scenario -- fixtures/raid-01.json --real --out fixtures/raid-01.transcript-real.json
```

The API key must live at `~/.airaider/openai.env` (mode 600).
